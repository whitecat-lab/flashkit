# FlashKit

`FlashKit` is a native macOS app for creating bootable USB media and writing raw/appliance images, with a strong focus on Windows installer USBs and practical Linux/appliance support.

This repository is a Swift Package Manager macOS app. The package, executable target, and packaged app are all named `FlashKit`.

## Current Scope

- Windows installer USB creation with FAT32 extraction, WIM splitting when required, NTFS helper paths, patching, verification, and structural validation
- Raw image restore and capture for `.img`, `.raw`, `.img.gz`, `.img.xz`, `.img.bz2`, plus `VHD`/`VHDX` workflows
- Linux/appliance-aware inspection and strategy selection for:
  - Proxmox VE
  - TrueNAS
  - OpenWrt
  - OPNsense
  - pfSense
- FreeDOS media creation
- Persistent activity logging with backend diagnostics

## Requirements

- Apple Silicon Mac
- macOS 15 or newer
- Xcode command line tools / Swift 6.3 toolchain

## Repo Layout

- `Sources/`: app, helper protocol, and privileged helper source
- `Resources/`: app icon, FreeDOS assets, UEFI:NTFS image, bundled helper payloads
- `script/`: packaging, helper bundling, and privileged-helper install scripts
- `Tests/`: backend and app tests

## Build

```bash
swift build

```

## Run The App

To build the app bundle and launch it:

```bash
./script/build_and_run.sh --verify
```

This produces:

- `dist/FlashKit.app`

## Install The Privileged Helper

FlashKit prefers a launchd-installed privileged helper for smoother destructive writes, cleaner worker attribution, and better progress reporting.

If the helper is missing or mismatched, FlashKit can still use the macOS administrator password prompt when a privileged write or capture needs elevation.

For contributors and local testers, helper installation is recommended one-time setup when you want to exercise the helper-backed path explicitly or avoid repeated password prompts during destructive operations.

Install it once:

```bash
./script/install_privileged_helper.sh
```

Remove it:

```bash
./script/uninstall_privileged_helper.sh
```

The install script builds the helper as your normal user first, then prompts for `sudo` only for the `/Library` install and `launchctl` bootstrap step.

## Bundled Helpers

`FlashKit` bundles several helper binaries inside the app under `Contents/Resources/Helpers`. These are used for image handling, filesystems, decompression, and Windows-media workflows.

Examples currently bundled include:

- `wimlib-imagex`
- `qemu-img`
- `mkntfs`
- `ntfsfix`
- `mke2fs`
- `xz`
- `ntfs-populate-helper`
- `freedos-boot-helper`

The helper manifest is generated by:

- `script/bundle_helpers.sh`
